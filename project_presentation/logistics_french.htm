Setting Up Configuration
There are many ways to organize the source code for your TinyBase system. Il ya plusieurs façons d'organiser le code source de votre système TinyBase. Many of you will be tempted to break your source code up across directories -- for instance one directory for the RM component, one for IX, and so on. Beaucoup d'entre vous seront tentés de briser votre code source en le plaçant à travers les répertoires - par exemple un répertoire pour le composant RM, l'un pour IX, et ainsi de suite. However, our experience dictates that for building the system and for TA grading, it is vastly preferable that you keep all of your TinyBase source code in a single directory. Please structure your code management in this way throughout the project. Cependant, notre expérience indique que pour la construction du système de classement et d'assistance technique, il est  préférable que vous conserviez tous vos codes sources du projet TinyBase dans un seul répertoire. S'il vous plaît, structurez la gestion de votre code de cette façon tout au long du projet.

To get you started, we are providing the Paged File ( PF ) component of the TinyBase system. Pour vous aider à démarrer, nous vous fournissons le composant  fichier paginé (PF)  du système TinyBase. The functionality of this component is described in detail in the PF Component document . La fonctionnalité de ce composant est décrit en détail dans le document composante PF . Later we will be providing a few additional routines and a command-line parser. Plus tard, nous offrirons quelques routines supplémenatires et un analyseur en ligne de commande.

We have written a script that automates the process of getting all necessary files to support your TinyBase code development. Nous avons écrit un script qui automatise le processus d'obtention de tous les fichiers nécessaires pour soutenir le développement de votre code TinyBase. You will need to invoke the script once for each of the four basic components of the TinyBase system: each invocation retrieves the files needed for that component. Vous aurez besoin d'invoquer le script une fois pour chacun des quatre composants de base du système TinyBase: chaque invocation récupère les fichiers nécessaires pour ce composant. The first invocation will fetch the entire PF component, template rm.h and rm_rid.h files, and an RM "tester" file ( rm_test.cc ). Le premier appel va chercher le  composant PF en entier, les fichiers d'entête rm.h  et rm_rid.h, et un fichier de test pour le composant RM(rm_test.cc). Here is the sequence of steps you need to follow. Voici la séquence des étapes que vous devez suivre.

  1. Login and create a directory for the TinyBase system: Connectez-vous et créez un répertoire pour le système TinyBase: 
      mkdir tinybase tinybase mkdir
  2. Change into the directory: Allez dans le répertoire: 
      cd tinybase tinybase cd
  3. To make sure that nobody else can read the directory: Pour vous assurer que personne d'autre ne peut lire le répertoire: 
      fs la . fs la.
  4. Make a symbolic link to the TinyBase setup script by typing: Faire un lien symbolique vers le script d'installation TinyBase en tapant: 
      ln -s /usr/class/cs346/TinyBase/scripts/setup . ln-s /cal/homes/adieng/public_html/TinyBase/script/scripts/setup.
  5. To get information about the setup script execute: Pour obtenir des informations sur le script d'installation, exécuter: 
      ./setup -h . / Setup -h
  6. Now execute the setup script. Maintenant, exécuter le script d'installation. It will transfer or create symbolic links to all files necessary for you to complete the first project part. Elle va transférer ou créer des liens symboliques vers tous les fichiers nécessaires dont vous aurez besoin pour compléter la première partie du projet. You can use the " -v " switch in order to see exactly what is happening. Vous pouvez utiliser l'option "-v" pour voir exactement ce qui se passe. Don't forget to include the argument " 1 ". Ne pas oublier d'inclure l'argument "1". 
      ./setup 1 . / Setup 1
  7. The setup script will create a src directory. Le script d'installation va créer un répertoire src. Change into the src directory: Allez dans le répertoire src: 
      cd src cd src
  8. An initial Makefile is provided for you. Un Makefile initial est prévu pour vous. You should use this makefile as a basis for the makefile that will manage all of your TinyBase source code. Vous devez utiliser ce makefile en tant que model de base pour le makefile qui va gérer l'ensemble de votre code source TinyBase. Typing " make " alone will create the library for the PF component. «Make» à elle seule créer la bibliothèque de la composante PF. Typing " make testers " will create the PF library and three PF component test programs. En tapant "make testeurs" créer la bibliothèque de PF et trois éléments de  test PF. Type: Tapez: 
      make testers make testeurs
    pf_test1 , pf_test2 , and pf_test3 are executables that test the PF component. pf_test1, pf_test2, et pf_test3 sont des exécutables permettant de tester la composante PF. Feel free to try out the tests if you want. Also, you can use these tests as inspiration for other test you will create. N'hésitez pas à essayer les tests si vous voulez. De plus, vous pouvez utiliser ces tests comme source d'inspiration pour les autres tests que vous allez créer.
You will also need to get supporting files for the next component. Vous aurez aussi besoin d'obtenir les fichiers de support pour le composant suivant. Use the same setup script, but substitute as an argument to the script the correct component number: use " 2 " for IX, " 3 " for SM, and " 4 " for QL. Utilisez le même script de configuration, mais en substituant l'argument du script le nombre correspondant au composants : utilisez "2" pour IX, "3" pour SM, et "4" pour QL.

Grading Le classement
In general, you will find that it is quite difficult to get perfect scores on all of the components. En général, vous trouverez qu'il est assez difficile d'obtenir des notes parfaites sur tous les composants. A perfect score requires bullet-proof code, good documentation, and smart design choices. Un score parfait nécessite l'épreuve du code de balle, une bonne documentation, et les choix de conception intelligente. Each of the first four components of the TinyBase project -- the RM, IX, SM, and QL components -- will be graded out of 40 points. Chacune des quatre premières composantes du projet TinyBase - la RM, IX, SM, et des composants QL - seront notés sur 40 points. For most components a score above 35 is considered very solid work. Pour la plupart des composants un score supérieur à 35 est considéré comme un travail très solide. The scoring for these components is broken into four main parts: La notation de ces composants est divisé en quatre parties principales:

Functionality Fonctionnalité

Correct functionality means that your program passes all of the tests in the TA's test suite. Corriger fonctionnalité signifie que votre programme passe tous les tests de la suite de test de la TA. Your score on functionality will be directly proportional to the number of tests passed. Votre score sur la fonctionnalité sera directement proportionnel au nombre de tests réussis. Some of the tests will try to break your code, so don't be surprised if we uncover bugs you didn't think about. Certains de ces tests va essayer de casser le code, alors ne soyez pas surpris si nous découvrons les bugs que vous ne pensez pas à. We provide some sample tests, but these tests are generally much simpler than the "best" of the TA's tests. It is your responsibility to fully test your component (see Testing above). Nous fournissons quelques exemples de tests, mais ces tests sont généralement beaucoup plus simple que "le" meilleur des TA teste les. Il est de votre responsabilité de vérifier pleinement votre appareil (voir essai ci-dessus).

Documentation Documentation

As mentioned above, all students must submit a 1-2 page description of your design, key data structures, and testing strategy for your component. Comme mentionné ci-dessus, tous les étudiants doivent soumettre une page de description de 1-2 de votre conception, structures de données clés, et la stratégie de tests pour votre composant. This document should focus on any novel features in your implementation, and on the modularity you should have used in coding the component (see Modularity/Correctness below). In addition, we expect that you will sufficiently comment your code. Ce document devrait mettre l'accent sur les nouvelles caractéristiques de votre application, et sur la modularité, vous devriez avoir utilisé dans le codage de la composante (voir Modularité / Correction ci-dessous). En outre, nous espérons que vous serez suffisamment commenter votre code. By "sufficiently" we mean that it will be possible for the TA to read a small, specific part of the code and understand how it works. Par «suffisamment» nous voulons dire qu'il sera possible pour le TA de lire une petite partie spécifique du code et de comprendre comment il fonctionne. As a rule of thumb, a comment on every line is too much, while large chunks of code with no comments at all is too little. En règle générale, un commentaire sur chaque ligne est trop, alors que de gros morceaux de code avec aucun commentaire à tous, c'est trop peu. More details are provided in the RM Component document . Plus de détails sont fournis dans le composant de document RM .

Design Choices Les choix de conception

Within each component there are certain design choices that you will need to make. Au sein de chaque composante il ya des choix de conception certain que vous aurez besoin de faire. Usually, some of the options require less work and result in less efficient code. Habituellement, quelques-unes des options nécessitent moins de travail et produire un code moins efficace. Other options are more work but result in code that is more stable, faster, or results in less wasted space. For each component the TA will choose some number of design issues that were left unspecified in the component description. D'autres options sont plus de travail mais le résultat dans le code qui est plus stable, plus rapide, ou les résultats dans l'espace gaspillé moins. Pour chaque composante de la TA va choisir un certain nombre de problèmes de conception qui ont été laissées non spécifiées dans la description du composant. You will receive an email message within a few days of submitting the component, asking you to describe how you implemented those aspects of the component. Vous recevrez un message e-mail dans quelques jours la présentation de la composante, vous demandant de décrire comment vous avez réalisés ces aspects de la composante. Your descriptions should be at most 1-2 paragraphs long, and they should include references to relevant file names and line numbers within those files. Votre description doit être plus à 1-2 de longs paragraphes, et ils devraient inclure des références à des noms de fichiers et les numéros de ligne dans ces fichiers. Based on your response -- which the TA will verify by looking at your code -- you will be assigned a "design choices" score. Sur la base de votre réponse - que le TA sera de vérifier en regardant votre code - vous sera attribué un «choix de conception" score. The score will be based on the choices that you made, whether your code properly implements those choices, and the clarity and accuracy of your description. Le score sera basé sur les choix que vous faites, si votre code implémente correctement ces choix, et de la clarté et la précision de votre description. You will have 48 hours to send a reply to the TA's questions; failure to respond within this time will result in a zero score in the design choices category. Vous avez 48 heures pour envoyer une réponse à la TA questions; l'absence de réponse dans ce délai se traduira par un score de zéro dans les choix de la catégorie de conception.

Modularity/Correctness Modularité / Correction

An elegant or modular design of the code is a somewhat subjective measure. Un design élégant ou modulaire du code est une mesure subjective peu. Most approaches will earn full credit. La plupart des approches accumule un crédit complet. There are, of course, some very bad designs that will lose points. Il ya, bien sûr, d'autres mauvaises conceptions qui va faire perdre des points. In addition, a component can pass all of the functionality tests but not be implemented correctly. En outre, un élément peut passer tous les tests de fonctionnalité, mais ne pas être mis en œuvre correctement. For example, a one-megabyte memory leak is considered an incorrect implementation. Par exemple, un mégaoctet de fuite de mémoire et un est considéré comme une transposition incorrecte.
