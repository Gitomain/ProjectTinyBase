Master DataScale - Bases de Données Avancées - Projet TinyBase


Le projet TinyBase est divisé en quatre parties:

  1. Composante Record Manager (RM): Dans cette partie vous devez implanter un ensemble de fonctions pour gérer les fichiers non-ordonnés (heap) de records (tuples). Cette composante s'appuiera sur le module Page File (PF) que nous fournissons. Le module PF gère les opérations "bas niveau" dans un fichier, à la granularité des pages (voir aussi la documentation associée, en anglais).

  2. Composante Indexage (IX): Dans cette composante vous allez implanter les fonctionnalités nécessaires pour la création d’indexes sur des fichiers non-ordonnée. L’index utilisera les arbres B +. Cette composante s'appuiera sur le module PF.

  3. Composante System Manager (SM): Dans cette composante vous allez implanter différentes fonctionnalités liées aux commandes DDL et au catalogue du système. La composante SM s'appuiera sur les modules précédents, RM et IX. On utilisera également un parseur pour des instructions DDL en ligne de commande, que nous fournissons.

  4. Composante Query Language (QL): Dans cette partie vous allez réaliser toutes les fonctionnalités nécessaires à l’exécution de requêtes SQL simples (fragment RQL ; requêtes et mises à jour). Cette composante s'appuiera sur les trois composantes précédentes et utilisera le parseur en ligne de commande que nous fournissons.



Nous vous fournirons le code pour le module Page File (PF) ainsi que pour certaines routines utilisées couramment dans d'autres composantes.<\cr> Nous allons également fournir un parseur en ligne de commande que vous utiliserez pour les parties 3 et 4 du projet.

Les spécifications nécessaires sont données pour le code que nous fournissons, ainsi que les spécifications pour chaque composante que vous allez implanter. Ces spécifications seront données comme interfaces OO (langage C++). Nous allons vous aider à démarrer votre projet en fournissant des exemples de makefiles, des fichiers header (les point-h), etc. En outre, pour certaines parties du projet, nous fournirons des suites de tests avant la date d'échéance, même si ces tests ne seront pas exhaustifs.

Sessions d'aide    

Il y aura une session d'aide quelques jours avant chaque date d’échéance. Le but des séances d'aide est de discuter les décisions de conception pour chaque partie du projet et de répondre aux questions les plus fréquentes. Les séances d'aide ne sont pas obligatoires.

Machines Linux

Vous allez implanter TinyBase en sachant que votre code sera testé en utilisant les machines Linux en salle C133. Les séances d’aide se dérouleront en salle C133. Pour les instructions concernant la compilation et les tests voir le document de logistique.
