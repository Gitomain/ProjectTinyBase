Project Overview Aperçu du projet

The TinyBase project is divided into four parts: Le projet TinyBase est divisé en quatre parties:

  1. The Record Management (RM) Component: In this part you will implement a set of functions for managing unordered files of database records. La gestion des dossiers (RM) Composant: Dans cette partie vous pourrez mettre en œuvre un ensemble de fonctions pour gérer les fichiers non-ordonnée des documents de base de données. This component will rely on a Paged File (PF) component that we will provide. Ce volet s'appuiera sur un fichier paginé (PF) composant que nous fournissons. The Paged File component performs low-level file I/O at the granularity of pages. Le fichier de composant paginé effectue niveau fichier de bas I / O à la granularité des pages.

  2. The Indexing (IX) Component: In this part you will implement a facility for building indexes on records stored in unordered files. L'indexation (IX) Composant: Dans cette partie vous pourrez mettre en œuvre une installation pour créer un index sur les dossiers stockés dans des fichiers non-ordonnée. Your indexing facility will be based on B+ trees. The Indexing component will rely on the Paged File component. Votre installation d'indexation sera basée sur des arbres B +. Le composant d'indexation s'appuiera sur le fichier de composant paginé.

  3. The System Management (SM) Component: In this part you will implement various database and system utilities, including data definition commands and catalog management. Le système de gestion (SM) Composant: Dans cette partie vous mettra en œuvre diverses bases de données et le système de services publics, y compris la définition des commandes de données et de gestion de catalogue. The System Management component will rely on the Record Management and Indexing components from Parts 1 and 2. Le composant de gestion du système s'appuiera sur le gestion des dossiers et des composants d'indexation des parties 1 et 2. It also will use a command-line parser, which we will provide. Il sera également utiliser un analyseur en ligne de commande, qui nous fournira.

  4. The Query Language (QL) Component: In this part you will implement RQL -- the RedBase Query Language. Le langage de requête (QL) Composant: Dans cette partie vous mettrez en œuvre RQL - l'RedBase Query Language. RQL consists of user-level data manipulation commands, both queries and updates. RQL se compose de données au niveau des commandes de manipulation de l'utilisateur, les requêtes et les mises à jour. The Query Language component will rely on the three components from Parts 1-3, and it will use the command-line parser that we are providing. Le langage de requête composante s'appuiera sur les trois composantes de pièces 1-3, et il utilisera l'analyseur en ligne de commande que nous fournissons.

We will provide code for the Paged File (PF) component of RedBase and for some commonly-used routines in other components. Nous vous fournirons le code pour le fichier paginé (PF), composante de RedBase et pour certains-routines utilisées couramment dans d'autres composantes. We will also provide a command-line parser that you will use for Parts 3 and 4 of the project. Nous allons également fournir une ligne analyseur de commande que vous utiliserez pour les parties 3 et 4 du projet. Specifications for the code we provide, along with specifications for each component that you will implement, will be given as object-oriented interfaces in the C++ programming language. We will help you get started with your programming by providing sample Makefiles, header files, etc. In addition, for some of the project parts we will provide test suites in advance of the due date, although these tests will not be comprehensive. Spécifications pour le code que nous offrons, ainsi que les spécifications pour chaque élément que vous mettrez en œuvre, sera donné comme orientée interfaces d'objets dans le langage C + + de programmation. Nous allons vous aider à démarrer avec votre programmation en fournissant des Makefiles échantillon, les fichiers d'en-tête, etc . En outre, pour certaines parties du projet, nous fournirons des suites de tests à l'avance de la date d'échéance, même si ces tests ne sera pas complète.

Project Help Sessions Sessions d'aide projet

There will be a help session at 15:15 each Friday afternoon preceding the due-date for the parts of RedBase project. Il y aura une session d'aide à 15:15 chaque vendredi après-midi précédant la date d'échéance pour les parties du projet RedBase. The purpose of the help sessions is to discuss design decisions for each project part and to answer commonly-asked programming-related questions in a group setting. Le but des séances d'aide est de discuter les décisions de conception pour chaque partie du projet et de répondre plus fréquemment posées des questions liées à la programmation dans un contexte de groupe. The help sessions are not required, and good students should have no trouble completing the project without attending the help sessions. Les séances d'aide ne sont pas tenus, et les bons étudiants devraient avoir aucune difficulté à réaliser le projet sans assister aux sessions de l'aide.
